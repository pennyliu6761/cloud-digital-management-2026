### 💡 觀念講解：什麼是 CMS？Notion 為何能當網站？

**CMS（Content Management System，內容管理系統）** 是讓非工程師也能管理網站內容的工具。
```
傳統網站架構：
設計師設計 → 工程師寫程式 → 部署伺服器 → 內容更新還要再找工程師

CMS 網站架構：
你在 Notion 寫內容 → 一鍵發布 → 網站自動更新
```

| 工具 | 定位 | 適合情境 |
|------|------|---------|
| Notion Sites | 超簡單，無需設定 | 說明頁、文件、作品集 |
| GitHub + Vercel | 彈性高，可自訂 | 互動網頁、專題展示 |
| WordPress | 功能最完整 | 大型部落格、電商 |

> **本週學 Notion Sites（簡單版），Week 11 學 GitHub + Vercel（進階版）。**
> 兩個都學，讓你根據需求選擇最適合的工具。

### 💡 觀念講解：Notion Database 是什麼？

**Notion Database** 是 Notion 最強大的功能——
它同時是「結構化資料庫」和「可以切換多種視圖的展示工具」。
```
同一份資料，四種視圖：

Table View  → 像試算表一樣逐列顯示
Board View  → 像 Trello 一樣用看板分類
Gallery View → 像 Instagram 一樣用卡片展示
List View   → 簡潔的清單模式
```

> 這和 Google 試算表的最大差異：
> Sheets 只有一種視圖（表格），Notion Database 可以切換多種視圖，
> 適合對外展示給不同需求的使用者。

> [!NOTE]
> **嵌入 vs 連結的差異：**
> - **連結**：學生點擊後跳轉到 Google 表單頁面（離開你的網站）
